torrent 协议学习

2022-01-10 torrent, 种子,bt 评论

BitTorrent协议

BitTorrent协议（简称BT，俗称比特洪流、BT下载）是用在对等网络中文件分享的网络协议程序。
和点对点（point-to-point）的协议程序不同，它是用户群对用户群（peer-to-peer），而且用户越多，
下载同一文件的人越多，下载该文件的速度越快。且下载后，继续维持上传的状态，就可以“分享”，
成为其用户端节点下载的种子文件（.torrent），同时上传及下载

Torrent File

种子使用了一种名为Bencode 编码

Bencode 支持4中编码

  • Byte String
  • Integer
  • List
  • Dictionary

种子文件是一个Bencode编码的Dictionary

  • announce: 这是一个URL指向了tracker 服务器
  • info: 这是一个字段，包含文件相关的信息
    • lenght: 文件的总长度，单位是字节
    • name: 文件名
    • piece lenght: Piece(分段)的长度
    • Pieces: 一个数组里面对应了每个pieceDe SHA1哈希值,用于校验(每个固定长度为20)

如果是多个文件那么info 里面会有一个files 数组，分别存储每个文件的length 与path

Tracker

announce 指向了Tracker 服务器
Tracker 服务器不提供任何下载，只用来沟通peers,。
每个Peer 通过PeerId 来表示自己，这是一个20字节的数据，格式没有要求
我们可以通过Tracker 服务器获取当前资源的peer,同时可以注册自己为peer

Tracker 使用HTTP 协议，请求通过Query携带参数

  • info_hash: 用来请求资源的标识(在BT中对资源的唯一标识使用的是InfoHash,就是对种子资源里的info字段惊醒SHA1后得到的结果20字节)
  • peer_id: 生成自己表示身份的Id 20字节
  • port: 我们客户端监听的端口，用于监听其他peer 的消息

Tracker 返回的信息使用Bencode编码，里面含有两个数据 interval与peers

  • peers: 是一个Byte Array 6个字节代表一个peer 前4字节为地址，后2个位BigEndian 端口号

下载

下载流程

  1. 解析种子
  2. 请求Tracker，获取Peers列表
  3. 请求Peers 下载Piece根据Pieces 字段验证有效性
  4. 组装Piece

Peer 握手包

  • 1 字节的协议长度ProtocolLength 固定0x13
  • 19 字节的协议名 ProtocolName固定BitTorrent protocol
  • 8 字节的保留字段Reserved ,都填写0
  • 20字节的InfoHash 从种子计算获取
  • 20 字节的PeerId 自己生成

BTMessage

  • 4字节的Length,BigEndian
  • 1字节的ID,表名消息内容
  • X 字节的消息体PayLoad,含有具体数据 X为length - 1

消息类型

  • Choke:告诉对方不能请求任何数据，先等待
  • Unchoke: 告诉对方可以开始请求数据了
  • Have: 告诉对方我又某个Piece
  • BitField:将我又的所有Piece编码成Bitfield 发送给对方
  • Request: 向对方请求某个Piece
  • Piece: 发送Piece 数据给对方

我们连接到Peer时默认处于Choked状态，不允许向Peer 请求任何数据,直到Peer发送Unchocke 消息

使用 Request 下载时，并不是一次请求一个完整的 Piece，而是分为 Block 下载，Block 的大小可以在消息体中指定，一般为 16K。
